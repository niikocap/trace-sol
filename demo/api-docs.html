Rice Supply Chain API

Comprehensive API documentation for the Rice Supply Chain Management System

Base URL: http://localhost:3001/api/v1

üîê Authentication

All API endpoints require authentication using JWT (JSON Web Tokens). Include the token in the Authorization header of your requests.

Request Headers

Authorization: Bearer YOUR_JWT_TOKEN

Replace YOUR_JWT_TOKEN with your actual JWT token. Contact your administrator if you don't have a token.

üë• Chain Actors

5 endpoints

Manage farmers, millers, distributors, and other participants in the rice supply chain.

GET /chain-actors

Retrieve a paginated list of all chain actors with optional filtering.

Query Parameters

Parameter Type Description
page integer Page number (default: 1)
limit integer Items per page (default: 10, max: 100)
type string Filter by actor type (e.g., 'Farmer', 'Miller')

Example Request

curl -X GET \
  http://localhost:3001/api/v1/chain-actors \
  -H 'Authorization: Bearer YOUR_JWT_TOKEN'

Example Response (200 OK)

{
  "success": true,
  "data": {
    "actors": [{
      "publicKey": "...",
      "name": "John Doe",
      "actorType": ["Farmer"],
      "organization": "Organic Farms Inc.",
      "pin": "123456",
      "tps": "TPS-001",
      "isActive": true,
      "createdAt": "2023-01-01T00:00:00.000Z",
      "updatedAt": "2023-01-01T00:00:00.000Z"
    }],
    "total": 1,
    "page": 1,
    "limit": 10
  }
}

‚ö†Ô∏è Error Handling

The API uses standard HTTP status codes to indicate the success or failure of a request. In case of an error, the response will include a JSON object with error details.

Error Response Format

{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message",
    "statusCode": 400,
    "details": {
      // Additional error details (if any)
    }
  }
}

Common Error Codes

Status Code Error Code Description
400 BAD_REQUEST The request was invalid or cannot be served
401 UNAUTHORIZED Authentication required or invalid credentials
403 FORBIDDEN Insufficient permissions to access the resource
404 NOT_FOUND The requested resource was not found
422 VALIDATION_ERROR The request data failed validation
429 RATE_LIMIT_EXCEEDED Too many requests, please try again later
500 INTERNAL_SERVER_ERROR An unexpected error occurred on the server

Example Error Response

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validation failed",
    "statusCode": 422,
    "details": {
      "name": "Name is required",
      "email": "Invalid email format"
    }
  }
}
